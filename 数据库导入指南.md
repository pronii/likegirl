# LikeGirl 数据库导入指南

## 📋 准备工作

在开始之前，确保你已经：
- ✅ 安装了 phpstudy（包含 MySQL 和 Apache/Nginx）
- ✅ phpstudy 正在运行
- ✅ 代码已经在 `D:\phpstudy_pro\WWW\` 目录下

---

## 🗄️ 方法一：使用 phpMyAdmin（推荐新手）

### 步骤 1：启动 phpstudy 并打开 phpMyAdmin

1. 打开 phpstudy 软件
2. 确保 MySQL 服务已启动（绿色状态）
3. 点击 phpstudy 界面上的 **"数据库"** 或 **"phpMyAdmin"** 按钮
4. 或者直接访问：`http://localhost/phpmyadmin/`

### 步骤 2：登录 phpMyAdmin

- **用户名**: `root`
- **密码**: `root`（phpstudy 默认密码）

### 步骤 3：创建数据库

1. 点击左侧的 **"新建"** 或顶部的 **"数据库"** 标签
2. 在 "创建数据库" 输入框中输入：`likegirl`
3. 排序规则选择：`utf8mb4_general_ci`
4. 点击 **"创建"** 按钮

### 步骤 4：导入 SQL 文件

1. 点击左侧刚创建的 `likegirl` 数据库
2. 点击顶部的 **"导入"** 标签
3. 点击 **"选择文件"** 按钮
4. 找到并选择：`D:\phpstudy_pro\WWW\love_db.sql`
5. 其他选项保持默认
6. 滚动到页面底部，点击 **"执行"** 按钮
7. 等待导入完成（看到 "导入已成功完成" 提示）

### 步骤 5：验证导入结果

导入成功后，左侧应该显示以下数据表：
- ✅ about
- ✅ article
- ✅ diySet
- ✅ IPerror
- ✅ leavSet
- ✅ leaving
- ✅ login
- ✅ loveImg
- ✅ lovelist
- ✅ text
- ✅ warning

总共应该有 **11 张表**。

## 💻 方法二：使用 MySQL 命令行

### 步骤 1：打开命令行

1. 打开 phpstudy
2. 找到 MySQL 版本，点击旁边的 **"命令行"** 按钮
3. 或者手动进入 phpstudy 的 MySQL bin 目录

### 步骤 2：登录 MySQL

```bash
mysql -u root -p
```

输入密码：`root`

### 步骤 3：创建数据库

```sql
CREATE DATABASE likegirl CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 步骤 4：选择数据库

```sql
USE likegirl;
```

### 步骤 5：导入 SQL 文件

```sql
SOURCE D:/phpstudy_pro/WWW/love_db.sql;
```

⚠️ **注意**：路径使用正斜杠 `/`，不是反斜杠 `\`

### 步骤 6：验证导入

```sql
SHOW TABLES;
```

应该显示 11 张表。

### 步骤 7：退出

```sql
EXIT;
```

## ⚙️ 配置数据库连接

### 检查配置文件

打开 `D:\phpstudy_pro\WWW\admin\Config_DB.php`，确认以下配置：

```php
//数据库地址
$db_address = "localhost";

//数据库用户名
$db_username = "root";

//数据库密码
$db_password = "root";

//数据库名
$db_name = "likegirl";

//安全码（用于修改密码等敏感操作）
$Like_Code = "851055720ccZ@";
```

### 如果需要修改

如果你的 MySQL 配置不同，请修改对应的值：
- 数据库用户名不是 `root` → 修改 `$db_username`
- 数据库密码不是 `root` → 修改 `$db_password`
- 想用其他数据库名 → 修改 `$db_name`（同时创建数据库时也要用相同名称）

## ❓ 常见问题

### 问题 1：导入时提示 "文件太大"

**解决方案**：
1. 在 phpMyAdmin 中，点击 "导入" 标签
2. 查看 "上传文件的最大大小" 限制
3. 如果 `love_db.sql`（18KB）超过限制，需要修改 PHP 配置：
   - 打开 phpstudy → 设置 → PHP 设置 → php.ini
   - 找到 `upload_max_filesize` 和 `post_max_size`
   - 改为 `20M` 或更大
   - 重启 Apache/Nginx

### 问题 2：导入后无法连接数据库

**检查清单**：
- ✅ MySQL 服务是否启动（phpstudy 显示绿色）
- ✅ 数据库名是否正确（`likegirl`）
- ✅ 用户名密码是否正确（`root/root`）
- ✅ `Config_DB.php` 配置是否保存

### 问题 3：访问提示 "Cannot connect to database"

**原因**：数据库没有创建或配置文件错误

**解决**：
1. 登录 phpMyAdmin 检查 `likegirl` 数据库是否存在
2. 检查 `admin/Config_DB.php` 中的配置
3. 查看 MySQL 错误日志

### 问题 4：后台无法登录

**可能原因**：
- 数据库导入不完整
- `login` 表没有数据

**解决**：
1. 在 phpMyAdmin 中打开 `likegirl` 数据库
2. 点击 `login` 表
3. 检查是否有数据（用户名 `admin`）
4. 如果没有，重新导入 `love_db.sql`

### 问题 5：页面显示乱码

**原因**：字符编码不对

**解决**：
1. 确保数据库字符集是 `utf8mb4`
2. 在 phpMyAdmin 中：左侧点击数据库 → 操作 → 排序规则 → 选择 `utf8mb4_general_ci`
3. 保存并重新导入
