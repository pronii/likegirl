# 批量本地图片上传功能 - 使用说明

## 功能概述

新增了批量本地图片上传功能，可以一次性从电脑本地选择多个图片文件上传到服务器。

## 访问路径

**后台管理** → **恋爱相册** → **批量上传（本地文件）**

或直接访问：`/admin/batchUploadLocal.php`

## 功能特点

✅ **多文件选择**：支持一次性选择多个图片文件（按住 Ctrl/Cmd 多选）  
✅ **格式支持**：JPG、PNG、GIF、WEBP  
✅ **自动压缩**：原图自动压缩到 1920x1080，质量 85%  
✅ **自动缩略图**：自动生成 300x300 缩略图  
✅ **实时进度**：显示上传进度和每个文件的状态  
✅ **容错机制**：部分文件上传失败不影响其他文件  
✅ **预览功能**：选择后立即显示图片预览  
✅ **批量设置**：统一设置相册和日期  
✅ **单独描述**：每张图片可单独添加描述

## 使用步骤

### 1. 选择文件
- 点击"选择图片文件"区域
- 在弹出的文件选择器中选择图片
- 可按住 Ctrl（Windows）或 Cmd（Mac）多选文件
- 支持的格式：JPG、PNG、GIF、WEBP
- 单个文件最大 10 MB

### 2. 设置参数
- **所属相册**：选择图片要添加到的相册（可选）
- **日期**：设置图片的日期（必填，默认当天）
- **描述**：为每张图片单独添加描述（可选）

### 3. 开始上传
- 点击"开始上传"按钮
- 系统会逐个上传文件并显示进度
- 每个文件会显示状态：
  - ⏸️ 等待上传（灰色）
  - ⏳ 正在上传（蓝色，带旋转动画）
  - ✓ 上传成功（绿色）
  - ❌ 上传失败（红色，显示错误原因）

### 4. 查看结果
- 上传完成后显示统计：成功数、失败数
- 如果全部成功，2秒后自动跳转到图片管理页面
- 如果有失败，可以查看失败原因

## 文件限制

| 限制项 | 值 |
|--------|-----|
| 单文件最大 | 10 MB |
| 支持格式 | JPG、PNG、GIF、WEBP |
| 压缩后尺寸 | ≤ 1920x1080 |
| 缩略图尺寸 | 300x300 |
| 压缩质量 | 85% |

## 服务器要求

### PHP 配置（推荐）

编辑 `php.ini` 文件：

```ini
upload_max_filesize = 100M    ; 单个文件最大 100MB
post_max_size = 200M          ; POST 总大小最大 200MB
max_execution_time = 300      ; 脚本执行时间 5 分钟
memory_limit = 256M           ; 内存限制 256MB
```

### 必需的 PHP 扩展

- GD 库（用于图片处理）
- MySQLi（数据库操作）
- JSON（JSON 响应）

### 目录权限

确保以下目录可写：
- `/uploads/images/` - 存储压缩后的图片
- `/uploads/thumbs/` - 存储缩略图

## 常见问题

### Q1: 上传失败提示"文件过大"
**A:** 检查 PHP 配置中的 `upload_max_filesize` 和 `post_max_size` 是否足够大。

### Q2: 上传后没有显示在前台相册
**A:** 请确认选择了相册（不是"未分类"），未分类的图片仅在后台可见。

### Q3: 图片被压缩后质量下降
**A:** 这是正常现象。系统会自动压缩图片以节省存储空间和带宽。如需调整压缩质量，可以修改 `batchUploadLocalPost.php` 中的压缩参数。

### Q4: 部分文件上传失败
**A:** 查看失败原因：
- 文件格式不支持
- 文件大小超过限制
- 文件损坏或不是有效的图片
- 服务器磁盘空间不足

### Q5: 上传速度慢
**A:** 上传速度取决于：
- 网络带宽
- 图片文件大小
- 服务器性能
建议先压缩图片再上传，或分批上传。

## 安全性

✅ 会话验证 - 必须登录后才能访问  
✅ 文件类型检查 - 只允许图片格式  
✅ 文件大小限制 - 防止服务器资源耗尽  
✅ 真实性验证 - 使用 `getimagesize()` 验证是否为真实图片  
✅ 预处理语句 - 防止 SQL 注入  
✅ XSS 防护 - 使用 `htmlspecialchars()` 处理用户输入  
✅ 唯一文件名 - 使用时间戳 + uniqid() 防止文件名冲突

## 维护建议

1. **定期清理磁盘空间**  
   上传的图片会占用服务器空间，建议定期检查 `/uploads/` 目录。

2. **数据库备份**  
   上传的图片信息保存在 `loveImg` 表中，建议定期备份。

3. **日志监控**  
   后端会记录上传日志到 PHP error_log，可用于故障排查。

4. **性能优化**  
   如果上传量较大，可以考虑：
   - 使用 CDN 存储图片
   - 启用服务器端缓存
   - 优化数据库索引

# 相册封面选择功能

## 功能概述

在**新增相册**和**修改相册**时，封面支持三种选择方式：

1. **输入URL** - 手动输入外链图片地址（原有功能）
2. **上传图片** - 从电脑选择图片上传，自动压缩为封面（最大 800x800）
3. **从图库选择** - 直接选择已上传到服务器的图片作为封面

## 访问路径

**后台管理** → **恋爱相册** → **相册管理** → **新增相册 / 修改相册**

## 使用方法

1. 进入新增或修改相册页面
2. 在"相册封面"区域选择一个选项卡：
   - **输入URL**：粘贴图片链接
   - **上传图片**：点击上传区域，选择本地图片，自动上传并压缩
   - **从图库选择**：点击缩略图选中，再次确认即可
3. 选中后下方会显示"当前封面预览"
4. 如需取消，点击"清除封面"按钮
5. 保存相册即可

## 涉及文件

### 新增文件
- `admin/albumCoverPicker.php` - 可复用的封面选择器组件（三种方式）
- `admin/albumCoverUploadPost.php` - 封面上传后端（压缩到 800x800）
- `admin/getImageLibrary.php` - 图库数据接口（分页返回已上传图片）

### 修改文件
- `admin/albumAdd.php` - 引入封面选择器组件
- `admin/albumMod.php` - 引入封面选择器组件（带当前封面回显）

### 存储位置
- 上传的封面存放在 `/uploads/covers/` 目录
- 数据库 `love_album` 表的 `album_cover` 字段保存封面路径

## 技术说明

- 封面选择器是独立组件，新增和修改页面共用同一份代码
- 三种方式最终都写入隐藏字段 `name="album_cover"`，后端无需改动
- 图库选择复用现有 `loveImg` 表数据，优先显示缩略图
